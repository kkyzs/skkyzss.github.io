Regarding the second game, I've created a ping pong style game, where the player simply has to move the paddle and essentially play against an AI. I'm currently stuck with how I should equate the movement of the AI paddle. I've essentially equated the "y" value of the paddle to equal the "cy" value of the ball, so it is essentially faultless. I know because I actually played for 10minutes and never won a single round while testing different variations of the equation. A friend suggested that I use the random function to make it such that it's predetermined when the AI will make a mistake. This works but is not exactly what I wanted, since the the difference in movement is very obvious. My next idea is to essentially use a random number generator, so it adds a random value to the paddle's "y" value each time, giving variation to where the ball contacts the paddle, and when the generated number is high enough, the AI essentially makes a mistake and misses the ball. I've yet to try it out but in theory, this should work! I hope...?

While I was coding this game, the main issue I initially had (that's now resolved), was measuring the collision between a Raphael circle and rectangle. I thought it'd be easy since we did collisions between two circles in class, but I was so wrong. This was because the circle object already used "cx" and "cy", the coordinates in the middle of the object that made calculating the overlap easy, but rectangles didn't have the same thing. It's simple math really, but for some reason I struggled. After testing out multiple equations over a few days of ragequitting, I finally got the proper equation to work. Though currently, sometimes there seems to be an error with the overlap, and the ball essentially gets "stuck" within the paddle's borders, until it bounces out again. This is a rare occurence but I'd like to fix it eventually, though it's probably not a huge issue now. I really never thought I'd have to do so much math in coding.

Another issue I encountered was how I was going to increase the difficulty of the game as it progressed. I initially simply added a value to the x and y rates, but didn't work, as the values would swap between positive and negative with each hit, making it essentially a loop between two numbers, but not adding any to them. I considered using various Math() functions to resolve the issue, but none seemed to work either. I eventually settled on using multiplication instead, since this increases the values exponentially without having the sign be an issue. It wasn't what I planned, but turned out to be an effective way to resolve the issue.
