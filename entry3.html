One issue I encountered during styling was trying to make alert pop-up boxes my visually pleasing, unfortunately, the regular "alert()" commands are not going to work in this case. Instead, I created divs and styled them accordingly, and by using "display: block" and "display: none", I was able to toggle between showing and hiding them properly. Initially, when I first did this, I created multiple divs that essentially did the same thing but for separate text, such as victory box and death box, and manipulated them accordingly in Javascript as such. I created multiple divs in the HTMl because I initially didn't know how to change the onClick functions of the buttons when manipulated using ".innerHTML", but when coding the same pop-up boxes for the second game, I realised all I had to do was reassign the event listeners accordingly. While both methods work, the second is more compact and cleaner, so if there's extra time, I'll make the necessary changes.

Regarding the randomising of errors for the AI paddle, while my initial plan in the previous diary entry did work, because the random value added to the "y" value was being generated every 20ms (due to the setInterval() of the code), it essentially looked like the AI was having a seizure. So essentially, what I decided to do was to create a random number, which I did, but not every 20ms. So I decided to do it when the ball contacted the player's paddle, storing that number into a variable and adding it to the "y" value of the AI. Thankfully, this works! While it does mean that AI does "jump" a bit when the ball contacts the player's paddle, it's not a huge issue in my opinion.
